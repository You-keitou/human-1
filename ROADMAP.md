# Roadmap

M0〜M5(設計 → server → ui → cli → deploy・実機 E2E)は完了。ここからは「人間 LLM としての回答体験」と「訓練の多様性」を太くする。

## M6: Whiteboard 拡充

**ゴール**: draw.io を目指さない。「設計タスクの回答に出てくる図」を最短の手数で描けること。出力はこれまでどおり Mermaid テキストに変換してエディタへ挿入する(人間の最終出力はテキスト、という原則は維持)。

### 図種の追加(Mermaid 変換先があるものだけ)

- [ ] シーケンス図: ライフラインノード+メッセージエッジ(実線/破線、activate 区間)→ `sequenceDiagram`
- [ ] 状態遷移図: 状態ノード+遷移ラベル、開始/終了擬似状態 → `stateDiagram-v2`
- [ ] フローチャート図形: 判断(ひし形)・開始終了(角丸)・入出力 → `graph` の形状記法(`{}` `()` `[/ /]`)
- [ ] グループ/境界枠(サービス境界・BC 境界)→ `graph` の `subgraph`

### 作図の速さ(図種より効くかもしれない)

- [ ] エッジラベルのインライン編集(ダブルクリック)
- [ ] ノードのコピー/複製(⌘D)・複数選択の整列・スナップ
- [ ] undo / redo(⌘Z / ⇧⌘Z)
- [ ] ダブルクリックで空白にノード追加 → そのまま名前入力
- [ ] パレットのキーボードショートカット(付箋 `s`、サービス `v` など)

### 非ゴール

- 自由描画・任意図形・画像貼り付け(draw.io 化はしない)
- Mermaid → ホワイトボードの逆変換(需要が出たら stretch)

## M7: 出題者ペルソナ

**ゴール**: トレーナーに人格を持たせ、訓練の質感を選べるようにする。UI の Personas ページでカードを眺めて選ぶと、ローカルで叩くべき `hllm` コマンドが出てくる(トレーナーはローカル実行なので、UI は「コマンドの発行所」に徹する)。

### 仕組み

- [ ] ペルソナ定義を `packages/shared` に(id / 表示名 / 一言口上 / アイコン・色 / system prompt 断片 / 採点観点の重み)
- [ ] CLI: `hllm train "お題" --persona <id>`(トレーナーの system prompt に合成。`--persona` 未指定は現行挙動)
- [ ] UI: Personas ページ — カードギャラリー。クリックで詳細+実行コマンド表示(コピーボタン付き):
  `hllm train "システム設計" --persona ijiwaru --epochs 3`
- [ ] rollout / スコアに persona タグを永続化し、学習曲線をペルソナ別に見られるようにする

### 初期ペルソナ案

| id | 人格 | 採点の癖 |
| --- | --- | --- |
| `ijiwaru` | 少し意地悪な出題者。エッジケースと例外系を執拗に突く | 抜け漏れに厳しい。褒めない |
| `mentor` | 優しいメンター。ヒントを小出しにする | 改善の方向性を必ず添える |
| `interviewer` | 厳格な面接官。要件の言語化とトレードオフ説明を要求 | 「なぜ」の説明不足を減点 |
| `timekeeper` | 時間に厳しい。短い回答時間を宣言して急かす | 遅延・冗長さを減点 |
| `architect` | 抽象好きの設計者。境界と責務の切り方だけを見る | 図の構造で採点 |

### プロセス上の注意

- Personas ページの UI は Pencil モック(Style C)→ px 一致ゲートの通常フローに乗せる
- ペルソナ prompt は「採点タグ `[SCORE: x.x/10]` 必須」を壊さないこと(トレーナー共通制約に追記)
